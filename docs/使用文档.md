# aliyun-ims-mixcut 使用文档

这份文档面向第一次接入 `hunjian/aliyun-ims-mixcut` 的业务开发，目标是让你先跑通一次提交，再根据业务复杂度决定是直接使用内置模板，还是下探到底层 `Builder / Model` 自定义时间线。

如果你已经看过 [README.md](../README.md)，可以把这份文档理解为“接入版说明书”：

- `README.md` 更偏能力总览和示例集合
- 本文档更偏接入顺序、选型建议和业务建模方式

## 1. 这个包解决什么问题

这个包是一个面向业务项目的 PHP Composer 包，用来封装阿里云智能媒体服务 IMS 的 AI 混剪能力。它不只是帮你调一次 `SubmitMediaProducingJob`，而是把常见的业务接入方式拆成了几层：

- 模型层：`Timeline`、`VideoTrack`、`SubtitleTrack`、`OutputMediaConfig`
- Builder 层：`TimelineBuilder`、`SubtitleBuilder`、`AudioBuilder`、`EffectTrackBuilder`
- 模板层：`PortraitMixcutTemplate`、`AiNarrationImageVideoTemplate`、`BatchRandomMixcutTemplate`
- 业务 DTO 层：`Material`、`MaterialPool`、`ThemeConfig`、`BatchTaskListBuilder`
- 客户端/服务层：`ImsClientFactory`、`ImsJobClient`、`MediaProducingService`、`BatchProducingService`

这几层分别解决的问题是：

- 你不需要手拼一大坨 IMS 官方数组
- 你可以先用模板快速接入，再慢慢下探到底层
- 你可以把“素材中心”“主题配置”“批量任务”这些业务概念映射到包里的 DTO

## 2. 什么时候用模板，什么时候自己拼时间线

优先建议这样判断：

- 你只是想快速生成一个固定结构的视频：优先用内置模板
- 你已经有素材池，要做一批不同版本：优先用 `BatchRandomMixcutTemplate`
- 你需要强控制镜头、片段顺序、字幕绑定、局部特效：用底层 `Builder / Model`

一个简单的判断标准是：

- 如果你的需求能用“给模板喂上下文”描述清楚，用模板
- 如果你的需求已经变成“我要自己控制每个 clip 的时间范围、布局、特效、字幕绑定”，就不要再硬套模板

## 3. 接入前准备

### 3.1 安装包

```bash
composer require hunjian/aliyun-ims-mixcut
```

如果你要真实调用阿里云接口，还需要安装官方 SDK：

```bash
composer require alibabacloud/ice-20201109 alibabacloud/credentials darabonba/openapi
```

### 3.2 配置环境变量

建议通过环境变量提供凭证和默认输出配置，不要把 AccessKey 硬编码进代码：

```dotenv
ALIYUN_IMS_ACCESS_KEY_ID=your-access-key-id
ALIYUN_IMS_ACCESS_KEY_SECRET=your-access-key-secret
ALIYUN_IMS_ENDPOINT=ice.cn-shanghai.aliyuncs.com
ALIYUN_IMS_REGION_ID=cn-shanghai
ALIYUN_IMS_BUCKET=your-output-bucket
ALIYUN_IMS_OUTPUT_PATH_PREFIX=mixcut/output
ALIYUN_IMS_PROJECT_ID=your-ims-project-id
ALIYUN_IMS_OUTPUT_MEDIA_TARGET=oss-object
```

`ImsConfig::fromEnv()` 会自动读取这些变量。

### 3.3 先理解一个重要默认行为

`ImsClientFactory::createAdapter($config)` 会自动判断走哪个适配器：

- 已安装官方 SDK，且 `AccessKeyId / AccessKeySecret / Endpoint / RegionId` 都有值：走 `OfficialIceAdapter`
- 否则：走 `StubAdapter`

这意味着：

- 本地写代码、看 JSON 结构时，不一定需要真实调用阿里云
- 但如果你以为自己在提交真实任务，结果却拿到的是 stub 返回值，通常就是依赖或环境变量没配全

## 4. 10 分钟跑通第一个任务

第一次接入时，建议先跑通一个最简单的本地模板提交流程，不要一上来就自己拼复杂时间线。

下面这个示例使用 `PortraitMixcutTemplate` 提交一个“横转竖 + 字幕 + BGM + 水印”的任务：

```php
<?php

require __DIR__ . '/../vendor/autoload.php';

use Hunjian\AliyunImsMixcut\Client\ImsClientFactory;
use Hunjian\AliyunImsMixcut\Client\ImsJobClient;
use Hunjian\AliyunImsMixcut\Config\ImsConfig;
use Hunjian\AliyunImsMixcut\Service\MediaProducingService;
use Hunjian\AliyunImsMixcut\Template\PortraitMixcutTemplate;

$config = ImsConfig::fromEnv();
$adapter = ImsClientFactory::createAdapter($config);
$client = new ImsJobClient($config, $adapter);
$service = new MediaProducingService($client);

$template = new PortraitMixcutTemplate();

$job = $service->submitLocalTemplate($template, array(
    'mainVideo' => 'oss://demo-bucket/materials/main.mp4',
    'bgm' => 'oss://demo-bucket/materials/bgm.mp3',
    'watermark' => 'oss://demo-bucket/materials/logo.png',
    'subtitleSegments' => array(
        array(
            'text' => '第一句字幕',
            'start' => 0.0,
            'end' => 2.0,
            'referenceClipId' => 'main-video',
        ),
        array(
            'text' => '第二句字幕',
            'start' => 2.0,
            'end' => 4.0,
            'referenceClipId' => 'main-video',
        ),
    ),
));

echo 'JobId: ' . $job->getJobId() . PHP_EOL;
echo 'Status: ' . $job->getStatus() . PHP_EOL;
```

这一条链路里最值得记住的是：

- `ImsConfig` 负责运行时配置
- `ImsClientFactory` 负责选择官方适配器或 stub 适配器
- `ImsJobClient` 负责把 `Timeline` / `OutputMediaConfig` 规范化成请求载荷
- `MediaProducingService` 是业务代码最常接触的门面

如果这一步能跑通，再去接你的业务素材和模板上下文。

## 5. 推荐接入路径

第一次接入时，不建议直接从底层 `TimelineBuilder` 开始。推荐按下面顺序选型。

### 5.1 场景一：单个主视频，想快速做横转竖

使用 `PortraitMixcutTemplate`。

适合：

- 一个主视频做横转竖
- 想加字幕、BGM、水印
- 想做模糊背景
- 想做单画面或三分屏布局

常用上下文字段：

- `mainVideo`
- `duration`
- `layout`
- `splitVideos`
- `blurRadius`
- `bgm`
- `watermark`
- `subtitleSegments`
- `globalFilter`
- `theme`
- `outputMediaConfig` 或 `outputMediaURL`

最常见的业务接法是：

- 你的业务系统传一个主视频 URL
- 如果需要三分屏，再传 `splitVideos`
- 如果字幕已经由业务端切好，直接传 `subtitleSegments`

### 5.2 场景二：图片和视频混排，加 AI 配音和字幕

使用 `AiNarrationImageVideoTemplate`。

适合：

- 图片和视频混编
- AI 配音
- AI_ASR 自动字幕
- 手工字幕和重点高亮
- 讲解型短视频

示例：

```php
<?php

use Hunjian\AliyunImsMixcut\Model\ThemeConfig;
use Hunjian\AliyunImsMixcut\Template\AiNarrationImageVideoTemplate;

$theme = (new ThemeConfig('narration-brand'))
    ->setVoice('zhitian_emo')
    ->setBgm('oss://demo-bucket/materials/bgm-soft.mp3')
    ->setWatermark('oss://demo-bucket/materials/logo.png')
    ->setGlobalFilter('warm')
    ->setGlobalVfx('glow')
    ->setHighlight(array(
        'Color' => '#F8E16C',
        'BorderColor' => '#1D3557',
    ))
    ->setSubtitleStyle(array(
        'font' => 'Alibaba PuHuiTi 2.0',
        'fontSize' => 50,
        'fontColor' => '#FFFFFF',
        'boxColor' => '#101820',
    ));

$template = new AiNarrationImageVideoTemplate();
$built = $template->build(array(
    'theme' => $theme,
    'materials' => array(
        array('type' => 'image', 'url' => 'oss://demo-bucket/materials/poster-1.jpg', 'duration' => 2.8),
        array('type' => 'video', 'url' => 'oss://demo-bucket/materials/clip-1.mp4', 'duration' => 3.2, 'transition' => 'fade'),
        array('type' => 'image', 'url' => 'oss://demo-bucket/materials/poster-2.jpg', 'duration' => 2.6),
    ),
    'narrationText' => '这是一条图片加视频混合的 AI 讲解短视频，由 AI 配音并自动生成字幕。',
    'manualSubtitleSegments' => array(
        array(
            'text' => '支持 AI 配音、ASR 字幕和重点高亮',
            'start' => 0.0,
            'end' => 3.0,
            'referenceClipId' => 'scene-0',
        ),
    ),
));
```

这类模板的关键是：

- `materials` 决定画面顺序
- `narrationText` 决定 AI 配音内容
- `manualSubtitleSegments` 是手工字幕补充，不等同于 ASR 自动字幕
- 模板内部会给每个片段设置 `scene-0`、`scene-1` 这种 `ClipId`

### 5.3 场景三：一个主题下批量出很多变体

使用 `BatchRandomMixcutTemplate`，配合 `MaterialPool`、`StrategyBuilder`、`BatchTaskListBuilder`。

适合：

- 同一活动要生成多条不同版本视频
- 你已经有素材池
- 想对片段顺序、时长、滤镜、转场、字幕样式做随机化

示例：

```php
<?php

use Hunjian\AliyunImsMixcut\Builder\BatchTaskListBuilder;
use Hunjian\AliyunImsMixcut\Builder\StrategyBuilder;
use Hunjian\AliyunImsMixcut\Model\Material;
use Hunjian\AliyunImsMixcut\Model\MaterialPool;
use Hunjian\AliyunImsMixcut\Model\ThemeConfig;
use Hunjian\AliyunImsMixcut\Template\BatchRandomMixcutTemplate;

$strategy = (new StrategyBuilder())
    ->transitions(array('fade', 'directional-left', 'directional-up'))
    ->filters(array('warm', 'cool', 'movie'))
    ->vfx(array('shake', 'glow', 'chromatic'))
    ->bgms(array(
        'oss://demo-bucket/materials/bgm-1.mp3',
        'oss://demo-bucket/materials/bgm-2.mp3',
    ))
    ->subtitleStyles(array(
        array('fontColor' => '#FFFFFF', 'boxColor' => '#000000'),
        array('fontColor' => '#F8E16C', 'boxColor' => '#1D3557'),
    ))
    ->clipDurations(array(2.5, 3.0, 3.5));

$pool = new MaterialPool();
$pool->add(Material::video('oss://demo-bucket/materials/v1.mp4', 10)->setSubtitle('视频素材 1'));
$pool->add(Material::video('oss://demo-bucket/materials/v2.mp4', 9)->setSubtitle('视频素材 2'));
$pool->add(Material::image('oss://demo-bucket/materials/i1.jpg')->setSubtitle('图片素材 1'));
$pool->add(Material::image('oss://demo-bucket/materials/i2.jpg')->setSubtitle('图片素材 2'));

$theme = (new ThemeConfig('spring-campaign'))
    ->setWatermark('oss://demo-bucket/materials/logo.png')
    ->setGlobalFilter('warm')
    ->setGlobalVfx('glow')
    ->setBgmPool(array(
        'oss://demo-bucket/materials/bgm-1.mp3',
        'oss://demo-bucket/materials/bgm-2.mp3',
    ))
    ->setSubtitleStyle(array(
        'fontColor' => '#F8E16C',
        'boxColor' => '#1D3557',
    ))
    ->setOutputPattern('oss://demo-bucket/mixcut/examples/batch/{theme}-{n}.mp4');

$tasks = (new BatchTaskListBuilder())
    ->template(new BatchRandomMixcutTemplate())
    ->pool($pool)
    ->strategy($strategy)
    ->theme($theme)
    ->count(3)
    ->sceneCount(4)
    ->seed(20260411)
    ->build();
```

这里推荐把“批量生成”的职责拆开：

- `MaterialPool` 管素材
- `ThemeConfig` 管活动默认配置
- `StrategyBuilder` 管随机策略
- `BatchTaskListBuilder` 负责把这些东西拼成可提交任务

## 6. 业务系统里怎么建模

这个包最适合放在业务系统和 IMS OpenAPI 之间，作为一层“视频制作域模型”。

### 6.1 素材中心 -> `Material` / `MaterialPool`

如果你业务里已经有素材中心，可以这样映射：

- 视频素材 -> `Material::video($url, $sourceDuration)`
- 图片素材 -> `Material::image($url, $duration)`
- 音频素材 -> `Material::audio($url, $sourceDuration)`

常见建议：

- `sourceDuration` 存源文件真实时长，方便随机裁切
- `subtitle` 存素材默认文案，方便批量模板直接生成字幕
- `adaptMode` 存画面填充策略，例如 `Cover` / `Contain`

### 6.2 主题配置 -> `ThemeConfig`

如果你业务里有“活动主题”“投放主题”“品牌模板”，建议统一映射成 `ThemeConfig`。

它适合承载：

- 默认配音人声 `voice`
- 默认单条 BGM `bgm`
- BGM 池 `bgmPool`
- 默认水印 `watermark`
- 默认全局滤镜 `globalFilter`
- 默认全局特效 `globalVfx`
- 默认字幕风格 `subtitleStyle`
- 高亮字幕配置 `highlight`
- 输出路径模式 `outputPattern`

`ThemeConfig` 的一个好处是：`applyToContext()` 只会填充上下文里未显式传入的字段，不会粗暴覆盖业务侧已经指定的值。

### 6.3 批量任务计划 -> `BatchTaskListBuilder`

如果你业务里有“批量发布计划”“一组素材批量出视频”的场景，推荐不要直接写 `for` 循环拼上下文，而是使用 `BatchTaskListBuilder`。

它会负责：

- 给每条任务生成递增 `seed`
- 把 `MaterialPool` 转成模板可消费的 `pool`
- 把 `ThemeConfig` 注入上下文
- 根据 `outputPattern` 或 `ThemeConfig::outputPattern` 生成输出地址

其中输出路径支持占位符：

- `BatchTaskListBuilder::outputPattern()` 支持 `{n}`
- `ThemeConfig::setOutputPattern()` 支持 `{theme}` 和 `{n}`

## 7. 提交、查询和轮询结果

### 7.1 提交本地模板

业务代码里最常见的是：

```php
$job = $service->submitLocalTemplate($template, $context, $options);
```

适用于：

- 使用本包内置模板
- 使用你自己实现的 `TemplateInterface`

### 7.2 提交手工时间线

如果你自己通过 `TimelineBuilder` 或模型对象拼好了时间线：

```php
$job = $service->submitTimeline($timeline, $outputMediaConfig, $options);
```

### 7.3 查询作业

```php
$job = $service->getJob($jobId);

echo $job->getStatus() . PHP_EOL;
echo $job->getMediaUrl() . PHP_EOL;
```

`JobResult` 已经把不同响应里的常见字段做了归一化，最常用的是：

- `getJobId()`
- `getStatus()`
- `getMediaUrl()`
- `isFinished()`
- `isFailed()`

### 7.4 轮询直到结束

```php
$result = $service->waitUntilFinished($jobId, 5, 600);

echo $result->getJobResult()->getStatus() . PHP_EOL;
echo $result->getJobResult()->getMediaUrl() . PHP_EOL;
```

`waitUntilFinished($jobId, $intervalSeconds, $timeoutSeconds)` 的行为是：

- 任务成功结束：返回 `ProduceResult`
- 任务失败：抛 `JobFailedException`
- 超时：返回一个 `ProduceResult`，但此时状态不一定是完成态

## 8. 什么时候下探到底层 Builder / Model

下面几种情况，建议直接用底层 API，而不是继续往模板里塞上下文：

- 一个镜头里有多个视频素材，要精确控制每段时长
- 字幕需要精确绑定到某个片段
- 同时有局部特效和全局特效
- 需要自定义布局、ZOrder、裁剪范围、转场、KenBurns
- 你已经在做“时间线编排”，而不只是“给模板填参数”

## 9. 底层自定义示例：两个镜头，每个镜头两个视频素材，多段字幕

这是业务里很常见的一类需求：不是一个主视频，而是自己组织镜头和字幕。

这个包里真正的组织方式是：

- 每个视频素材 = 一个 `VideoTrackClip`
- 每条字幕 = 一个 `SubtitleTrackClip`
- 字幕通过 `referenceClipId` 绑定到某个片段

示例：

```php
<?php

use Hunjian\AliyunImsMixcut\Builder\SubtitleBuilder;
use Hunjian\AliyunImsMixcut\Builder\TimelineBuilder;
use Hunjian\AliyunImsMixcut\Model\OutputMediaConfig;
use Hunjian\AliyunImsMixcut\Model\VideoTrack;
use Hunjian\AliyunImsMixcut\Model\VideoTrackClip;

$timelineBuilder = TimelineBuilder::make()->portrait(1080, 1920);

$videoTrack = (new VideoTrack())
    ->setMainTrack(true)
    ->setTrackShortenMode('ShortenFromEnd');

$clip1 = VideoTrackClip::fromMediaUrl('oss://bucket/scene1-a.mp4')
    ->setClipId('scene-1-a')
    ->setSourceRange(0, 3.0)
    ->setTimelineRange(0.0, 3.0)
    ->setLayout(0, 0, 1080, 1920, 'Cover');

$clip2 = VideoTrackClip::fromMediaUrl('oss://bucket/scene1-b.mp4')
    ->setClipId('scene-1-b')
    ->setSourceRange(0, 3.0)
    ->setTimelineRange(3.0, 6.0)
    ->setLayout(0, 0, 1080, 1920, 'Cover');

$clip3 = VideoTrackClip::fromMediaUrl('oss://bucket/scene2-a.mp4')
    ->setClipId('scene-2-a')
    ->setSourceRange(0, 3.0)
    ->setTimelineRange(6.0, 9.0)
    ->setLayout(0, 0, 1080, 1920, 'Cover');

$clip4 = VideoTrackClip::fromMediaUrl('oss://bucket/scene2-b.mp4')
    ->setClipId('scene-2-b')
    ->setSourceRange(0, 3.0)
    ->setTimelineRange(9.0, 12.0)
    ->setLayout(0, 0, 1080, 1920, 'Cover');

$videoTrack
    ->addClip($clip1)
    ->addClip($clip2)
    ->addClip($clip3)
    ->addClip($clip4);

$subtitleTrack = (new SubtitleBuilder())
    ->style('Alibaba PuHuiTi 2.0', 50, '#FFFFFF')
    ->box('#000000', 0.35, 20)
    ->outline('#111111', 4)
    ->shadow('#000000', 3, 3)
    ->layout(80, 1500, 920, 220, 'BottomCenter')
    ->autoWrap(true)
    ->fixedFontSize(true)
    ->buildTrack(array(
        array('text' => '镜头一 字幕1', 'start' => 0.0, 'end' => 1.5, 'referenceClipId' => 'scene-1-a'),
        array('text' => '镜头一 字幕2', 'start' => 1.5, 'end' => 3.0, 'referenceClipId' => 'scene-1-a'),
        array('text' => '镜头一 字幕3', 'start' => 3.0, 'end' => 6.0, 'referenceClipId' => 'scene-1-b'),
        array('text' => '镜头二 字幕1', 'start' => 6.0, 'end' => 7.5, 'referenceClipId' => 'scene-2-a'),
        array('text' => '镜头二 字幕2', 'start' => 7.5, 'end' => 9.0, 'referenceClipId' => 'scene-2-a'),
        array('text' => '镜头二 字幕3', 'start' => 9.0, 'end' => 12.0, 'referenceClipId' => 'scene-2-b'),
    ));

$timeline = $timelineBuilder
    ->addVideoTrack($videoTrack)
    ->addSubtitleTrack($subtitleTrack)
    ->build()['timeline'];

$outputMediaConfig = OutputMediaConfig::oss('oss://bucket/output/custom-timeline.mp4')
    ->setSize(1080, 1920);
```

这里最容易错的点是：

- `ClipId` 是视频片段自己的标识
- `referenceClipId` 是字幕关联到哪个片段
- 如果一条字幕跨越了素材切换点，最好拆成两条字幕

## 10. 全局效果和局部效果怎么分

接入时经常会混淆这两个层次：

- 局部效果：挂在 `VideoTrackClip::Effects` 或 `AudioTrackClip::Effects`
- 全局效果：放进 `EffectTracks`

一般可以这样记：

- 只影响某个片段的，放 clip 上
- 影响整条视频视觉风格或固定位置元素的，放全局 effect track

常见归类：

- 局部：`Transition`、`Filter`、`VFX`、`Crop`、`KenBurns`
- 全局：背景图、水印、全局滤镜、全局特效

如果把全局水印做成片段局部效果，后续随机换素材时很容易错位。

## 11. 输出地址如何确定

输出地址有三种常见来源，优先级从高到低通常是：

1. 你显式传入了 `outputMediaConfig`
2. 你传了 `outputMediaURL`
3. `ImsJobClient` 发现 `OutputMediaConfig` 里没 `MediaURL`，但配置了 `ALIYUN_IMS_BUCKET`，会自动按默认规则生成一个 OSS 地址

默认自动生成的文件名形态类似：

```text
ims-mixcut-YYYYMMDDHHIISS-xxxxxxxx.mp4
```

所以如果你希望业务侧能稳定追踪输出文件，建议：

- 单任务：自己传 `OutputMediaConfig::oss(...)`
- 批任务：通过 `BatchTaskListBuilder::outputPattern()` 或 `ThemeConfig::setOutputPattern()` 显式控制

## 12. 常见问题

### 12.1 为什么我明明调用了提交接口，却像没真正发到阿里云

先检查两件事：

- 是否安装了官方 SDK 依赖
- `AccessKeyId / AccessKeySecret / Endpoint / RegionId` 是否都存在

如果不满足，`ImsClientFactory` 会回退到 `StubAdapter`。

### 12.2 为什么提交成功了却没有 `JobId`

`ImsJobClient` 在提交后会把返回值映射为 `JobResult`。如果返回结构里没有它能识别的 `JobId` 字段，会抛出 `JobSubmitException`。

通常说明：

- 适配器返回结构和预期不一致
- 你自己扩展了适配器，但没有按 `JobResult::fromSubmitResponse()` 可识别的字段返回

### 12.3 为什么字幕和画面对不上

重点检查：

- 字幕的 `start / end` 是否对应时间线时间，而不是源素材时间
- `referenceClipId` 是否指向了正确的 `ClipId`
- 字幕是否跨越了片段切换点

### 12.4 为什么最终输出地址不是我想要的路径

重点检查：

- 是否显式传了 `outputMediaConfig`
- 是否设置了 `outputPattern`
- 是否只是依赖了 `ALIYUN_IMS_BUCKET + OUTPUT_PATH_PREFIX` 的默认生成行为

### 12.5 为什么报时间线无效

`ImsJobClient` 会先检查时间线至少包含一条视频轨道或音频轨道。如果两者都没有，会抛出 `InvalidTimelineException`。

### 12.6 ASR 高亮字段为什么偶尔需要调整

`AI_ASR` 和高亮字幕字段会受阿里云租户版本或接口细节影响。这个包已经预留了 `extraFields/raw` 和 `HighlightConfig` 一类扩展入口，但如果你碰到官方字段名变动，仍然需要按实际接口微调。

## 13. 推荐的接入落地方式

如果你是第一次在业务系统里接这个包，建议按下面顺序推进：

1. 先用 `PortraitMixcutTemplate` 或 `AiNarrationImageVideoTemplate` 跑通单任务
2. 再把业务素材映射成 `Material` / `MaterialPool`
3. 再把活动或品牌配置映射成 `ThemeConfig`
4. 最后再接 `BatchTaskListBuilder` 做批量化
5. 真正遇到复杂编排需求时，再下探到底层 `TimelineBuilder`

这样接入的好处是：

- 早期成本低
- 业务代码不会一开始就和底层时间线结构强耦合
- 后面需要高级编排时，也有平滑下探路径
